- 함께 공부하면 좋은 내용 변수 스코프에 관하여 
var:
var 키워드로 선언된 변수는 함수 스코프를 갖습니다. 함수 스코프란 변수가 선언된 함수 내에서만 유효하며, 함수 외부에서는 접근할 수 없습니다.
블록 스코프를 무시하고 함수 전체에서 유효하기 때문에 호이스팅(hoisting)이 발생합니다. 호이스팅은 변수 선언을 현재 스코프의 최상위로 끌어올리는 동작을 말합니다.
이러한 이유 때문에 var 보단 let or const 사용을 권장한다.


let 변할 수 있는 가능성이 있는 데이터를 저장하는데 유용함 
age , location

PI , birthday

- % 연사자로 할 수 있는 많은 것들 

function checkOddEven(number) {
  if (number % 2 === 0) {
    console.log(number + "은(는) 짝수입니다.");
  } else {
    console.log(number + "은(는) 홀수입니다.");
  }
}

// 예시
checkOddEven(10); // 10은(는) 짝수입니다.
checkOddEven(7);  // 7은(는) 홀수입니다.


- null 과 undefined 의 차이 

undefined 는 변수를 선언하고 값을 할당하지 않으면 해당 변수는 자동으로 undefined로 초기화

null은 개발자에 의해 명시적으로 값이 없음을 나타내기 위해 사용됩니다.
